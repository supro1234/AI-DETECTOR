⚠ Face Swap Detection

🔍

POINT 16 — FACE SWAP FORENSIC SCAN

The engine now runs a dedicated sub-analysis (Forensic Point 16) specifically targeting face swap artifacts. This is in addition to all existing 15 forensic checks. When indicators are detected, a prominent "FACE SWAP DETECTED" alert appears in the UI with confidence score.

Boundary Seam Analysis

Detects unnatural blending lines, halos, and color discontinuities along the face boundary, hairline boundary, and neck junction — the primary artifacts of face swap compositing.

Skin Tone Mismatch

Compares sub-surface scattering and skin hue between the face, neck, ears, and hands. Skin swap operations frequently leave detectable chrominance discrepancies.

Texture Inconsistency

A face with lower resolution/texture detail than the surrounding body is a strong face swap indicator. The engine checks texture frequency mismatch between facial and body regions.

Lighting Discrepancy

Detects when the light hitting the face originates from a different angle or temperature than the body illumination — a classic tell when compositing two separate source images.

Jawline & Ear Continuity

Checks for blending artifacts at the junction of the swapped face and the original ears or jaw. Unnatural blur isolation (blurry face on a sharp body) is also flagged.

Post-Processing Guard

The post_process_face_swap() function in analyze.py also scans all red flags and the face swap analysis text for known face swap keywords, providing a secondary detection pass.

Installation Guide

Standard deployment workflow for modern workstation environments.

1. Machine Preparation

Ensure Node.js (v18+) and Python (3.10+) are installed. If you face execution errors on Windows PowerShell, bypass terminal policy:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

2. Dependency Provisioning

# Root folder (Node.js — Express API + Vite frontend)
npm install

# Backend Python engine — use requirements.txt for exact versions:
pip install -r backend/requirements.txt

# Or install manually:
pip install google-generativeai groq requests python-docx Pillow

3. Neural Launch

Start the forensic lab in dev mode. Concurrently boots both Express and Vite servers.

npm run dev

4. API Key Setup

  • Gemini (Enterprise): Visit aistudio.google.com/apikey — paid tier recommended for production.
  • Groq: Visit console.groq.com — free tier available, excellent for speed.

Troubleshooting

  • ECONNREFUSED: Ensure your backend is running on localhost:8301. Check terminal for Node.js errors.
  • API Auth Failed: Tokens are case-sensitive. Verify provider selection (Gemini vs Groq vs OpenRouter).
  • Gemini 429 / Rate Limit: The engine automatically retries 3× with exponential backoff (1s → 2s → 4s), then falls back to OpenRouter if a key is available. Watch backend console for [GEMINI_RATE_LIMIT] messages. For persistent rate limits, upgrade to a paid Google AI Studio tier.
  • Face Swap Not Detected: The engine only flags face swap when CONCRETE evidence exists. If you believe a swap is present, try running in Gemini-only mode for highest detection accuracy.
  • Python Not Found: If python fails, update server.js line 14 to 'python3' for macOS/Linux users.
  • Upload Limit: Maximum file size is 50MB. High-res TIFFs should be converted to JPG/PNG for faster analysis.
  • Report Download Fails: Ensure python-docx and Pillow are installed: pip install python-docx Pillow
